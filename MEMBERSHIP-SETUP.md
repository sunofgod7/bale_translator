# راهنمای تنظیم چک عضویت کانال

## مشکل: دکمه "تایید عضویت" کار نمی‌کند

این مشکل معمولاً به این دلیل است که ربات نمی‌تواند لیست اعضای کانال را بررسی کند.

## راه‌حل 1: اضافه کردن ربات به عنوان ادمین کانال (توصیه می‌شود)

### مراحل:

1. **به کانال خود بروید** (`@motarjem_mehran`)

2. **به تنظیمات کانال بروید**
   - روی نام کانال کلیک کنید
   - گزینه "مدیران" یا "Administrators" را انتخاب کنید

3. **ربات را به عنوان ادمین اضافه کنید**
   - روی "افزودن ادمین" کلیک کنید
   - username ربات خود را جستجو کنید
   - ربات را انتخاب کنید

4. **دسترسی‌های لازم را بدهید**
   - حداقل این دسترسی لازم است:
     - ✅ **دسترسی به پیام‌ها** (Access to messages)
   - سایر دسترسی‌ها اختیاری هستند و می‌توانید آن‌ها را غیرفعال کنید

5. **ذخیره کنید**

### تست کنید:

```bash
# شناسه کاربری خود را از @userinfobot دریافت کنید
node test-membership.js YOUR_USER_ID
```

اگر همه چیز درست باشد، باید پیام "✅ کاربر عضو کانال است!" را ببینید.

---

## راه‌حل 2: غیرفعال کردن موقت چک عضویت (برای تست)

اگر نمی‌خواهید ربات را ادمین کنید یا فقط می‌خواهید ربات را تست کنید:

### در Railway:

1. به داشبورد Railway بروید
2. به تنظیمات پروژه بروید
3. به بخش "Variables" بروید
4. یک متغیر جدید اضافه کنید:
   - **Name:** `ENABLE_MEMBERSHIP_CHECK`
   - **Value:** `false`
5. ذخیره کنید (Railway به صورت خودکار دوباره دیپلوی می‌کند)

### محلی (Local):

فایل `.env.local` را ویرایش کنید:
```env
ENABLE_MEMBERSHIP_CHECK=false
```

⚠️ **هشدار:** با این روش، همه کاربران می‌توانند از ربات استفاده کنند بدون اینکه عضو کانال باشند.

---

## راه‌حل 3: استفاده از Chat ID به جای Username

اگر روش‌های بالا کار نکرد، می‌توانید از شناسه عددی کانال استفاده کنید:

### پیدا کردن Chat ID کانال:

```bash
node test-membership.js
```

در خروجی، شناسه کانال را خواهید دید (مثلاً: `6387259492`)

### تغییر کد:

در فایل `api/webhook.js`، خط زیر را پیدا کنید:
```javascript
const REQUIRED_CHANNEL = "@motarjem_mehran";
```

و آن را به این تغییر دهید:
```javascript
const REQUIRED_CHANNEL = "6387259492"; // یا هر شناسه‌ای که دریافت کردید
```

## بررسی لاگ‌ها

برای دیدن دقیق مشکل، لاگ‌های Railway را بررسی کنید:

1. به داشبورد Railway بروید
2. به پروژه خود بروید
3. تب "Deployments" را باز کنید
4. روی آخرین deployment کلیک کنید
5. تب "Deploy Logs" را ببینید

وقتی روی دکمه "تایید عضویت" کلیک می‌کنید، باید لاگ‌هایی شبیه این ببینید:

```
[callback] Received callback_query: verify, userId: 123456789, chatId: 123456789
[callback] Verifying membership for user 123456789
[bale] getChatMember response for user 123456789: {"ok":true,"result":{"status":"member",...}}
[bale] User 123456789 status in channel: member
[callback] Membership check result: true
```

اگر خطایی وجود داشته باشد، در لاگ‌ها نمایش داده می‌شود.

## سوالات متداول

### Q: چرا ربات باید ادمین باشد؟
**A:** API بله نیاز دارد که ربات ادمین باشد تا بتواند متد `getChatMember` را فراخوانی کند و وضعیت عضویت کاربران را بررسی کند.

### Q: آیا ربات می‌تواند پیام‌های کانال را ببیند؟
**A:** خیر، ربات فقط می‌تواند بررسی کند که آیا یک کاربر خاص عضو کانال است یا نه. نمی‌تواند پیام‌ها را بخواند مگر اینکه دسترسی خاصی به آن بدهید.

### Q: آیا می‌توانم چک عضویت را کاملاً حذف کنم؟
**A:** بله، می‌توانید `ENABLE_MEMBERSHIP_CHECK=false` را تنظیم کنید یا کد مربوط به چک عضویت را از فایل `api/webhook.js` حذف کنید.
